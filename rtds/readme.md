# RTDS — Real-Time Data Server

Сервер сбора производственных данных в режиме реального времени с уровня автоматизации.   Предназначен для агрегации данных из различных источников, их временного хранения и предоставления для последующей обработки (например, сервером Historian).

---

## 📥 Источники данных

RTDS поддерживает следующие источники:
- **PLC** по протоколу **Modbus TCP** (MWP)
- **OPC UA/DA** серверы *(в планах)*
- **Реляционные базы данных** (через ODBC/SQL-коннекторы — *в разработке*)
- **MQTT/REST API** — *перспектива*

---

## 🏗 Архитектура

Сервер построен по модульной архитектуре с централизованным управлением и распределёнными коннекторами.

```
+------------------+
| Central Server   |
| (main process)   |
+--------+---------+
         |  ←→ Управление, передача тегов
         |
+--------v---------+     +------------------+
| Connector Proc   | ... | Connector Proc   |
| (Modbus TCP)     |     | (OPC, etc.)      |
+------------------+     +------------------+
         |
         |
+--------v---------+
| Local Buffer     |
| (SQLite)         |
+------------------+
         |
         ↓
     Historian
```

### Основные компоненты

| Модуль | Назначение |
|-------|-----------|
| `app.py` | Основной сервер: управляет процессами, очередями, тегами |
| `api/server.py` | REST API для мониторинга, управления и отладки |
| `storages/sqlite.py` | Локальное хранение данных в SQLite с фильтрацией дублей |
| `models/tag.py` | Модель тега: имя, значение, статус, временная метка |
| `configs/file.py` | Чтение конфигурации из файла (например, `config.ods`) |
| `connectors/` | Модули подключения к источникам данных |
| `scripts/` | Пользовательские скрипты для расчётов и преобразований |
| `config.ods` | Тестовая конфигурация в формате OpenDocument Spreadsheet |

## 🔌 Коннекторы

Каждый коннектор работает в **отдельном процессе** и запускается по расписанию (например, каждые 1–10 секунд).

### Функции коннектора:
- Подключение к источнику данных
- Чтение значений по заданным адресам
- Передача данных в центральный сервер
- Приём команд на запись от сервера
- Запись данных в источник (обратно)

### Формат строки конфигурации:

```Python
connector=ModbusTCP; host=192.168.1.100; port=502; unit_id=1; timeout=5; auto_open=true; auto_close=true
```

Каждый тип коннектора имеет свои параметры. Например, OPC UA требует url, namespace, security_mode.


Поддерживаемые коннекторы:

✅ ModbusTCP — Modbus TCP over IP

🟡 OpcUaConnector — планируется

🔲 SqlConnector — планируется

🧠 Скрипты (расчёты)

Скрипты позволяют выполнять промежуточную обработку данных после чтения и перед записью.

Когда выполняются:
- После завершения чтения всех коннекторов и перед началом записи в источники

Пример скрипта (scripts/calc_pressure.py):
```Python
import random as rnd

# Чтение тега
tgv = self.server.get('Pressure_Main')
print(f"Текущее значение: {tgv.value}")

# Генерация или расчёт
tgv.value = rnd.uniform(80.0, 120.0)
tgv.status = 0  # 0 = OK
tgv.timestamp = self.now()  # текущее время

# Сохранение
self.server.set(tgv)
```

Скрипты могут работать с массивами, триггерами, выполнять фильтрацию, усреднение и т.д.


### 💾 Локальный буфер
Для временного хранения данных используется SQLite.

Особенности, данные хранятся в двух таблицах:
- current_values — текущие значения тегов (перезаписываются)
- archive — архив с изменениями

При изменении значения более чем на заданное отклонение (deadband) — создаётся запись в архив.

Если значение не меняется в течение заданного интервала (max_age) — оно всё равно сохраняется в архив (anti-drift).
Historian забирает данные и сообщает временную метку, до которой можно очистить архив.

Пример стратегии:
```Python
deadband = 0.5   # минимальное изменение для записи
max_age = 300    # принудительная запись каждые 300 сек
```

## 📊 Метрики
Для мониторинга работы RTDS собираются ключевые метрики:

| Метрика |	Единица | Описание |
|-|-|-|
| connector_cycle_time_avg | сек |Среднее время выполнения цикла коннектора |
| buffer_write_speed |	tag/сек |	Скорость записи в архивный буфер |
| tags_total |	шт |	Общее количество активных тегов |
| connectors_running |	шт	| Количество запущенных коннекторов |
| script_execution_time	| мс |	Время выполнения скриптов |

Метрики публикуются через внутреннюю очередь и доступны через REST API.


### 🚀 Запуск сервера
```Bash
cd rtds
python3 app.py
```
Предварительные требования:
- Python 3.9+
- Библиотеки: pymodbus, openpyxl, sqlite3

Установка зависимостей:
```Bash
pip install pymodbus openpyxl
```

### 🔧 Разработка и тестирование

Конфигурация
Файл config.ods содержит:
- Список тегов
- Адреса в источниках
- Параметры коннекторов
- Настройки deadband, скриптов

Запуск в режиме отладки
```Bash
python app.py --debug --interval=5
```

### REST API (доступно при запуске).

- GET /api/status — текущий статус сервера
- GET /api/tags — список всех тегов
- GET /api/metrics — метрики производительности
- POST /api/reload — перезагрузка конфигурации
